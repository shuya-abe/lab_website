Linuxコマンド早見表（Raspberry Pi実習版）

概要

Raspberry Pi実習でよく使うLinuxコマンドを、用途別にまとめたページです。CUI初心者が「何を使えばよいか」を素早く判断できるように構成しています。

手順

1. まず環境確認コマンドを実行する

  1. whoami（ユーザ）
  2. hostname（接続先）
  3. pwd（現在地）
  4. ls -la（一覧）

2. 目的に応じてコマンド群を使い分ける

  1. ファイル確認: cat, less, head, tail
  2. 検索: find, grep
  3. ネットワーク確認: ip a, ping
  4. リソース確認: df -h, free -h

3. 実行後に必ず結果を確認する

  1. エラー文を読む。
  2. 期待したファイルや出力があるか確認する。
  3. 必要なら履歴（history）で振り返る。

コード

# ===== 基本確認 =====
whoami                 # コメント: 現在ユーザを確認
hostname               # コメント: 接続先マシン名を確認
pwd                    # コメント: 現在ディレクトリ
ls -la                 # コメント: 隠しファイル込み一覧

# ===== ディレクトリ・ファイル操作 =====
cd ~                   # コメント: ホームに移動
cd ..                  # コメント: 1つ上のディレクトリへ
mkdir -p work/class7   # コメント: ディレクトリをまとめて作成
touch sample.txt       # コメント: 空ファイル作成
cp sample.txt sample.bak   # コメント: コピー
mv sample.bak backup.txt   # コメント: 移動/リネーム
rm backup.txt          # コメント: ファイル削除（慎重に）

# ===== 内容確認 =====
cat sample.txt         # コメント: 全文表示
head -n 5 sample.txt   # コメント: 先頭5行
tail -n 5 sample.txt   # コメント: 末尾5行
less sample.txt        # コメント: ページ送り表示（qで終了）

# ===== 検索 =====
find . -name "*.py"     # コメント: 現在配下のPythonファイルを検索
grep -n "GPIO" led.py   # コメント: GPIOを含む行を行番号付きで検索

# ===== ネットワーク =====
ip a                   # コメント: IPアドレス確認
ping -c 3 8.8.8.8      # コメント: 疎通確認（3回だけ送信）

# ===== リソース確認 =====
df -h                  # コメント: ディスク使用量
free -h                # コメント: メモリ使用量

# ===== プロセス確認 =====
ps aux | grep python   # コメント: Python関連プロセス確認
top                    # コメント: リアルタイム監視（qで終了）

# ===== パッケージ管理 =====
sudo apt update        # コメント: パッケージ情報更新
sudo apt install -y tree  # コメント: パッケージ導入例

# ===== 便利操作 =====
history                # コメント: コマンド履歴確認
clear                  # コメント: 画面をクリア

解説

  • 迷ったら「環境確認 -> 操作 -> 結果確認」の順で進めます。
  • find と grep は、ファイル探索と文字列探索の基本セットです。
  • df -h と free -h は、実行失敗の原因切り分けで効果的です。
  • history はレポート作成時に実行手順を再現するのに役立ちます。

よくあるミス

  • rm を急いで実行し、復元できない削除を行う。
  • sudo を必要ない場面でも付けてしまう。
  • grep や find の検索範囲を広げすぎて、結果が読めなくなる。
  • top で終了方法（q）が分からず戸惑う。
