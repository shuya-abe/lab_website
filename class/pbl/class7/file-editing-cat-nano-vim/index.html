ファイル編集3手法（cat / nano / vim）

概要

Raspberry Piでよく使う3つの編集方法を比較します。短文メモは cat、初心者編集は nano、高速編集は vim が基本です。

加えて、実習でよく起こる「保存できていない」「別ファイルを編集していた」を防ぐ確認手順も扱います。

手順

0. 編集前の共通確認

  1. pwd で現在ディレクトリを確認する。
  2. ls -la で対象ファイルがあるか確認する。
  3. ファイル名は半角英数字と -、_ を中心に使う。

1. catで作成する

  1. cat > ファイル名 を実行する。
  2. 文章を入力する。
  3. Ctrl + D で保存終了する。
  4. cat ファイル名 で保存結果を確認する。

2. nanoで編集する

  1. nano ファイル名 を実行する。
  2. 入力・修正する。
  3. Ctrl + O で保存、Ctrl + X で終了する。
  4. cat ファイル名 で変更を確認する。

3. vimで編集する

  1. vim ファイル名 を実行する。
  2. i で入力モードへ入る。
  3. 編集後 Esc を押す。
  4. :wq で保存終了する。
  5. cat ファイル名 で変更を確認する。

4. バックアップを取りながら編集する（推奨）

  1. cp ファイル名 ファイル名.bak でバックアップを作成する。
  2. 編集する。
  3. 問題があれば mv ファイル名.bak ファイル名 で戻す。

コード

# 0) 事前確認
pwd
ls -la

# cat: 単発メモ向け
cat > note.txt
# コメント: 入力後に Ctrl + D で保存して終了
cat note.txt

# nano: 初心者向け編集
nano note.txt
# コメント: Ctrl + O で保存、Ctrl + X で終了
cat note.txt

# vim: 高速編集向け
vim note.txt
# コメント: i -> 入力、Esc -> ノーマル、:wq -> 保存終了
cat note.txt

# 失敗時に戻せるようにバックアップ
cp note.txt note.txt.bak

解説

  • cat は最短ですが、長文編集には不向きです。
  • nano は画面下にヘルプが出るため、最初の1本に向いています。
  • vim は慣れると高速ですが、モード切替の理解が前提です。
  • 初学者は「編集後に cat で確認」までを1セットにすると失敗が減ります。

3手法の比較

手法 向いている場面 学習コスト 編集しやすさ 注意点
cat 短いメモの新規作成 低い 低い 保存は Ctrl + D 必須
nano 初学者の通常編集 低い 中程度 終了時の保存確認に注意
vim 頻繁な編集・高速操作 高い 高い モード理解が必須

よくあるミス

  • vim で i を押さず、入力できないまま固まったと感じる。
  • vim で入力モードのまま :wq を打ってしまう。
  • cat 作成時に Ctrl + D を忘れて保存できない。
  • 同名ファイルが複数のディレクトリにあり、別ファイルを編集してしまう。
  • 上書き前にバックアップを取らず、元に戻せなくなる。

ミニトラブルシュート

  • nano: command not found が出る。
    • vim や cat を使うか、教員環境で nano 導入可否を確認する。
  • E45: 'readonly' option is set（vim）が出る。
    • 書き込み権限がない可能性がある。保存先と権限を確認する。
  • 文字化けする。
    • 文字コードや改行コードの差異を疑い、別エディタで確認する。
