Planner運用（開始日・期限・担当の管理）

概要

Plannerは、ToDoを書くだけでなく、開始日・期限・担当者をセットで管理して、作業の偏りを減らすために使います。

手順

1. ボードを準備する

  1. バケットを作る（例: 未着手、調査中、レビュー待ち、完了）。
  2. 最低6件のタスクを登録する。

2. タスク情報をそろえる

  1. 各タスクに担当者を設定する。
  2. 開始日と期限を設定する。
  3. 完了条件を説明欄に書く。

3. スケジュールを調整する

  1. カレンダー表示に切り替える。
  2. 同日に集中していないか確認する。
  3. 偏りがあれば開始日を調整する。

コード

# タスク記入ルール（コピペ用）
[タスク名] 既存事例を2件整理する
[担当者] 山田
[開始日] 2026-06-20
[期限] 2026-06-24
[完了条件] 2件の比較表を作成し、出典URLを記載する

# コメント:
# - 完了条件を具体化すると、終わったかどうかの判断がぶれない
# - 開始日を書くと、着手タイミングの先送りを防げる

解説

  • 期限だけだと直前に作業が集中しやすくなります。
  • 開始日まで入れることで、見通しが立ち、チームで早めに支援できます。
  • 完了条件が曖昧なタスクは、遅延と手戻りの原因になります。

よくあるミス

  • 「担当なし」「開始日なし」のタスクを作ってしまう。
  • タスクが大きすぎて、1回の作業で終わらない。
  • 完了条件がないため、レビュー時に合意できない。
